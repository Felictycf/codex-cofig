# Insight Architect Scoring Rubric

Use this rubric when rating profitability, relevance, and anti-fragility.

## Profitability Score (0-100)

Score formula:

`Profitability = Demand + Defensibility + Distribution + Execution + RiskAdj`

Subscores:

- `Demand` (0-25): market pull, urgency, repeat usage
- `Defensibility` (0-20): moat, switching cost, unique data/community edge
- `Distribution` (0-20): acquisition channels, reach compounding, conversion plausibility
- `Execution` (0-20): build/ops complexity, talent dependency, compliance friction
- `RiskAdj` (-15 to +15): downside asymmetry, platform/regulation volatility, fragility concentration

Guidance bands:

- `80-100`: Strong opportunity with clear path and manageable risk
- `60-79`: Viable but execution and differentiation decide outcome
- `40-59`: Weak economics or crowded market; only niche experiments
- `<40`: Low expected return relative to effort/risk

Confidence:

- `High`: multiple direct evidence points, recent data, low ambiguity
- `Medium`: some evidence but with notable gaps
- `Low`: mostly inferred, missing key verification data

## Relevance Mapping

Per track rating:

- `strong`: immediate actionable fit in current goals/workflows
- `moderate`: useful with adaptation or delayed timing
- `weak`: conceptually related but low current leverage

Priority rule:

- Prioritize tracks with `strong` relevance and `<= mid build` implementation effort.
- De-prioritize tracks with `weak` relevance and `heavy build` effort unless strategic.

## Anti-fragility Profile

Assess four dimensions, each 1-5:

- Platform dependence
- Regulatory/policy exposure
- Narrative dependency (hype sensitivity)
- Operational resilience (team/process/treasury durability)

Profile mapping:

- `fragile`: total <= 9
- `transitioning`: total 10-15
- `resilient`: total >= 16

## Bias Flags

Always check and report if observed:

- Survivorship bias
- Selection bias
- Engagement bias
- Story-over-data bias

Use neutral language. Flag risk without forcing a conclusion.
